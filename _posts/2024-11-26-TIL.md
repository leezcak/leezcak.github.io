SQL의 WHERE은 사실 '어디서'가 아니고 조건이다. 영어로는 말이 되지만 한국어로는 헷갈릴 수도 있겠다. FROM이 '어디서'고 WHERE은 '어떤' 이라는 뜻이다. 3년 동안 SQL을 배우면서 한번도 깊이 생각해 보지 않았는데 오늘 아티클 스터디에서 배웠다. 

내심 '코딩을 하면서 글까지 읽어야 돼? 대학교에서는 몇백페이지 리딩 하기 싫어서 챗지피티 돌렸었는데...' 라고 생각했는데 아티클들이 앞으로도 계속 이렇게 간결하고 straight to the point면 나쁘지 않겠다.

오늘은 코드카타를 몇개 풀고 (이걸 어떻게 제출하는 건지도 매니저님을 귀찮게 해서 알아냈다. 어떤 분은 벌써 몇십개 푸셨던데 나도 분발해야겠다.) 아티클 스터디를 끝내버리고 직무 스터디 피피티를 만들었다.

SQL을 5주차까지 들으려고 했는데 생각보다 시간이 많지 않아서 4주차까지만 듣고 코드카타를 더 풀어야겠다. 코드카타가 훨씬 SQL에 직접적으로 도움이 되는 것 같다.

제목은 LIKE 연산자다. 

```sql
SELECT *
FROM table
WHERE something_name LIKE 'A%' 
```

라고 한다면 something_name 중에 A로 시작하는 것을 모두 골라달라는 뜻이다. 

```sql
SELECT *
FROM table
WHERE something_name NOT LIKE 'A%' 
```

이거는 A로 시작하지 않는 모든 것들을 모두 골라달라는 뜻이고.

LIKE도 그렇고 %도 그렇고 뭔가 생긴 게 좀 귀엽다. 상해 완샹청에 %%카페가 있는데 한번도 못 가봤다. 텐퍼센트 카페라고 아무리 말해줘도 엄마는 이 카페를 응응 카페라고 부른다. 나중에 다시 갈 기회가 생기면 그땐 엄마랑 꼭 가야지.
